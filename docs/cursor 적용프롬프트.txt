docs/17-admin-content-management-spec.md를 콘텐츠 운영 관리 기준 문서로 등록한다.

향후 관리자 기능 개발 시 반드시 이 문서를 따른다.

관리 대상:

- Exam
- Subject
- Chapter
- Pattern
- Question
- AI Generated Content

모든 콘텐츠 변경은:

Draft
→ Review
→ Approved
→ Published

단계를 거친다.

콘텐츠 수정 시 Version과 변경 기록을 남긴다.

관리자 기능은 기존 학습 플랫폼 구조를 깨뜨리지 않고 별도 운영 레이어로 설계한다.


docs/18-ai-tutor-spec.md를 AI Tutor 기능 개발 기준 문서로 등록한다.

앞으로 AI 설명, 오답 분석, 추천 문제 생성 기능 개발 시 반드시 이 문서를 따른다.

AI Tutor는 단순 정답 제공 기능이 아니다.

반드시:

1. 사용자 답안 분석
2. 오답 유형 판단
3. Pattern 기반 설명
4. 수준별 해설
5. 추가 학습 추천

순서로 동작한다.

AI 생성 콘텐츠는 반드시 Question → Pattern → Solution 데이터와 연결한다.

근거 없는 문제 생성과 설명은 허용하지 않는다.

docs/19-performance-optimization-spec.md를 성능 최적화 기준 문서로 등록한다.

앞으로 기능 개발 시 성능 영향을 함께 검토한다.

원칙:

1. 필요한 데이터만 로딩한다.
2. 대용량 JSON은 분리한다.
3. LocalStorage에는 학습 상태만 저장한다.
4. 문제 전체 데이터 중복 저장을 금지한다.
5. 모바일 환경을 반드시 검증한다.
6. AI 호출은 필요한 경우만 수행한다.

현재 MVP 단계에서는 과도한 최적화를 하지 않고,
확장 가능한 구조를 유지한다.
